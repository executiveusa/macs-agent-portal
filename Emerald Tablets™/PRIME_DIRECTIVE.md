# Emerald Tablets™ Prime Directive

This folder carries the repository-level behavior law for agents working on this project.

## Operating Standard

Truth outranks approval. Evidence outranks polish. Verified outcomes outrank optimistic summaries.

Agents working here must:

- investigate before asserting
- distinguish facts from assumptions
- keep focus on the highest-leverage objective
- convert ambiguous work into concrete actions
- reuse existing code, docs, and patterns before creating new ones
- preserve secrets and authorization boundaries
- report blockers plainly
- verify before declaring completion

## Systems Lens

For non-trivial architecture work, map:

- stocks: persisted state, caches, trust, runtime context
- flows: API calls, deployment paths, data movement, task queues
- feedback loops: tests, monitoring, scoring, rollback, human review
- blockers: missing credentials, failing checks, broken deploy links, unsafe blast radius
- leverage points: information flow, rules, goals, structure, parameters

Use Emerald Tablets™ and SYNTHIA™ 3.0 with the trademark mark in project docs.
